# 사용자 흐름

## 앱 시작 시 게이팅
```
앱 실행
  └─ AuthContext mount: getSession() + onAuthStateChange 구독
       ├─ session 있음 → /(tabs)/closet
       └─ session 없음 → /sign-in
              └─ Google 로그인 → in-app browser → callback (PKCE) → setSession
                    └─ /(tabs)/closet 으로 이동
```

## 첫 실행 (로그인 후)
```
빈 옷장 → 중앙 CTA "옷을 등록해보세요"
```
온보딩 없음. 옷장 진입 시 Supabase 에서 자동 sync (push pending + pull remote).

## 옷 등록
```
FAB(+) → 바텀시트(카메라/갤러리)
→ 카메라: 1장 | 갤러리: 최대 5장
→ 로딩(스피너)
→ 분석 결과 카드 (모든 필드 수정 가능)
→ "저장하기" → 옷장 복귀
```

## 옷장 탐색
```
3열 그리드 (최신순)
카테고리 칩 탭 → 필터링
아이템 탭 → 상세/수정
```

## 옷 상세/수정
```
편집 가능한 폼 (등록과 동일 UI)
변경 시 "수정 완료" 활성
"삭제하기" → 확인 얼럿 → 옷장 복귀
```

## 피팅
```
피팅 탭 진입
├─ 전신 사진 없음 → 촬영 프롬프트 (실루엣 가이드)
└─ 전신 사진 있음 → 피팅 메인

피팅 메인:
  최근 피팅 썸네일 가로 스크롤 (탭 → 기록 보기)
  옷 그리드 체크박스 선택 + 카테고리 필터
  → "피팅해보기" → 로딩(스피너)
  → 결과: 합성 이미지 + [바꿔보기][저장]

바꿔보기 → 피팅 메인 (이전 선택 유지)
저장 → 로컬 + 최근 피팅에 추가
⚙️ → 전신 사진 재촬영
```

### 선택 규칙
- 카테고리당 최대 1개
- 원피스 ↔ 상의/하의 상호 배타
- 아우터로 레이어드 표현 (상의 + 아우터)

## 스타일 추천
```
추천 탭 → TPO 6개 프리셋 탭
→ 로딩(스피너)
→ 조합 카드 3개 (옷 썸네일 + 한줄 코멘트)
  "입어보기" → 피팅 메인 (옷 자동 선택, 사용자 확인 후 합성)
  "다시 추천받기" → 새 요청

캐싱 없음 — 매번 새 요청.
```

## 에러 케이스
- 서버 에러 → 토스트
- 빈 옷장 시 피팅/추천 → "옷을 먼저 등록해주세요" + 이동 버튼
- 옷장 부족 시 추천 → 가능한 만큼 + 안내 문구
